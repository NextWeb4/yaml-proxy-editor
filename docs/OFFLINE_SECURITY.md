# 离线与安全边界

## 本地优先

应用的默认状态是离线可用。打开、编辑、校验、格式化、结构识别、规则检查、DNS 审计、diff、备份和回滚都应在本地完成。

## 文件安全

- 打开文件必须由用户选择或显式输入路径。
- 保存文件前必须先创建备份。
- 备份目录位于用户数据目录下的 `backups`。
- 不允许自动扫描用户磁盘中的 YAML 文件。

## 配置隐私

以下数据不得上传：

- 本地 YAML 内容
- 节点服务器地址、端口、密码、UUID、密钥
- 订阅 URL
- 本地日志
- 备份文件

## 订阅隐私

订阅刷新只能访问用户主动提供的 URL。订阅 URL 应在 UI 和日志中脱敏展示。

## DNS / 泄露审计声明

DNS / fake-ip / IP 泄露审计模块只做配置风险提示和修改建议，不承诺绝对匿名，不替代真实网络环境检测。

`src/services/config/configOptimizer.ts` 的一键防泄露优化只改写本地 YAML，不访问泄露测试站点，不下载远程规则集。

## 依赖策略

- 不使用 CDN 加载前端资源。
- 不引入遥测 SDK。
- 不引入自动更新 SDK。
- 新依赖必须记录许可证、用途、替代方案和回滚方式。
